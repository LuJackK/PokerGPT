This project is a PokerGPT / nanoGPT-style Texas Hold'em next-action prediction experiment based on the project proposal and a very large PHH hand-history archive. Future agents should treat `poker-hand-histories.zip` as too large to extract: it is roughly 20 GB compressed and about 245 GB uncompressed, with hundreds of thousands of `.phh` / `.phhs` files, so all dataset work should use a streaming Python pipeline with `zipfile.ZipFile`, reading one member at a time, briefly parsing PHH/PHHS hands with PokerKit-compatible assumptions, and writing compact artifacts such as `train.bin`, `val.bin`, `meta.pkl`, and a manifest. The initial modeling target should be clean no-limit Texas Hold'em (`variant = 'NT'`) rather than mixing in the huge ACPC fixed-limit-heavy corpus; later work can optionally pretrain on broader poker grammar and fine-tune on no-limit human/Pluribus-style hands. Use a structured poker tokenizer instead of raw text, with tokens for hand/street/player/action/card/amount buckets, and avoid preserving noisy metadata like names, events, hand IDs, table names, or timestamps. For realistic decision modeling, train from the acting player's perspective: keep that player's private cards and public board cards, but mask unrevealed opponent hole cards as unknown. nanoGPT should be adapted rather than used as plain text generation: start small, use `block_size` around 128-256, mask loss toward decision/action tokens, add legal-action masking during generation, and consider separate heads for action type and bet/raise amount bucket. Evaluate with action accuracy, top-k accuracy, illegal move rate, street-specific accuracy, fold/check/call/bet/raise confusion, and amount-bucket error; illegal move rate is especially important because it shows whether the model learned poker legality rather than just token frequency.

Data selection plan: do not directly run preprocessing across the full zip on the first pass. First create a lightweight manifest by streaming only zip metadata and, when needed, the first small chunk or parsed header of each `.phh` / `.phhs` member; record member path, extension, compressed/uncompressed size, likely source folder, variant, betting structure, game type, player count, hand count estimate, and any parse errors. Use this manifest to select deliberate subsets such as `variant == NT`, no-limit Hold'em only, 6-max first, reasonable stack/action completeness, and non-corrupt files; keep ACPC fixed-limit-heavy folders out of the first training run unless doing optional grammar pretraining. The preprocessor should accept a manifest or selected-member list, then stream only those zip members into decision examples without extracting the archive. A practical sequence is: `build_manifest.py` writes `manifest.csv/jsonl`; `select_dataset.py` filters it into `selected_nt_6max.jsonl`; `prepare_poker.py --selection selected_nt_6max.jsonl` parses selected hands, creates one supervised example per decision point from the acting player's perspective, writes token IDs plus loss masks, and saves `train.bin`, `val.bin`, `meta.pkl`, and preprocessing statistics.
